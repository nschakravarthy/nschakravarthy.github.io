// agent harness engineering · a build log

Agent harness engineering, from the model up.

A model is a single component. Turning it into a dependable system means engineering everything around it. That means the context it reads, the tools it can call, the constraints it runs under, and the evaluation that catches regressions. I'm building that layer from first principles: starting at the model internals, and working up to autonomous agents that run in a loop and improve on their own runs.

Three layers, built from the model up. Each one sits on the one below it.

// the agent harness

When the model drives the loop

The same concerns as the harness, plus the ones that only appear once the model runs in a loop and chooses its own next step. Now the model controls the flow. So you need orchestration, state that survives across steps, hard limits, an audit trail, and a way for the system to learn from its own runs. This is where most of my attention is right now.

// keeping it running

Helix. Agent runtime

The plumbing that keeps an agent going even when something crashes. It writes down every step, so it can pick up exactly where it left off. Crashes are treated as normal, not exceptional.

// the part that matters most

Self-improving evaluation loop

This is the piece almost nobody builds. The system records every run, scores those runs, detects when quality regresses, and feeds targeted fixes back in. All of this happens while the underlying model stays fixed. The model holds constant; the system around it keeps getting better. If any of this is worth doing, it's this.
